## 一、项目概述

### 1.1 项目背景

随着城市生活节奏加快和居住空间限制，阳台种植成为越来越多都市人群亲近自然、放松身心的重要方式。然而，大多数阳台种植爱好者面临以下痛点：

**现状问题**：
- **知识匮乏**：缺乏系统的植物养护知识，不知道何时浇水、如何施肥
- **经验不足**：遇到病虫害无法识别，导致植物死亡
- **记录缺失**：没有良好的植物生长记录习惯，难以总结经验
- **交流困难**：身边缺乏同好，遇到问题无处咨询
- **方案通用**：现有APP提供的养护建议过于泛化，未考虑阳台特殊环境

**市场机会**：
- 中国阳台种植市场规模超百亿，且年增长率>20%
- 小程序生态成熟，用户使用门槛低
- AI技术成熟度提升，图像识别和大语言模型可商业化应用
- 社区+工具类产品粘性强，具备长期价值

### 1.2 产品定位

**产品名称**: PlantBrain（植物大脑）  
**产品形态**: 微信小程序（优先）+ 后续扩展H5/App  
**核心定位**: **专注阳台场景的AI驱动型植物全生命周期管理工具**

**价值主张**：
- 🌱 **个性化养护**：基于阳台环境和植物品种的定制化方案
- 🤖 **AI智能诊断**：拍照识别病虫害，智能问答解决养护难题
- 📊 **生长追踪**：完整的植物档案和时间轴记录
- 💬 **社区互助**：连接同好，分享经验，共同成长

### 1.3 目标用户

| 用户类型 | 占比 | 特征描述 | 核心需求 |
|---------|------|----------|----------|
| **新手小白** | 40% | 刚接触阳台种植，0-6个月经验 | 基础知识学习、操作简单易懂 |
| **进阶玩家** | 35% | 有一定经验，1-3年种植史 | 专业养护指导、问题快速诊断 |
| **资深达人** | 20% | 经验丰富，3年以上，可能有多盆植物 | 记录管理、知识分享、社交认同 |
| **园艺从业者** | 5% | 花店/园艺师等专业人士 | 工具效率、客户管理（潜在B端） |

### 1.4 产品目标

#### 短期目标（MVP发布后6个月）
- ✅ 注册用户数达到 **10,000+**
- ✅ 月活跃用户（MAU）达到 **3,000+**
- ✅ 用户人均添加植物 **≥3株**
- ✅ AI功能使用率 **≥40%**（注册用户中至少使用过一次）
- ✅ 次日留存率 **≥35%**

#### 中期目标（1年内）
- 🎯 注册用户数达到 **100,000+**
- 🎯 月活跃用户达到 **30,000+**
- 🎯 社区日均发帖量 **100+**
- 🎯 AI识别准确率（Top-3）**≥90%**
- 🎯 30日留存率 **≥25%**

#### 长期愿景（2-3年）
- 🌟 成为中国最大的垂直领域阳台种植社区
- 🌟 构建最完整的中文植物养护知识图谱
- 🌟 探索电商变现和SAAS服务模式

### 3.1 个人植物管理模块

#### 3.1.1 功能概述

为用户提供完整的植物档案管理能力，支持从添加植物到跟踪其整个生命周期的全过程记录。

#### 3.1.2 功能清单

| 编号 | 功能名称 | 优先级 | 复杂度 | 说明 |
|------|---------|--------|--------|------|
| P-001 | 添加植物 | P0 | 中 | 支持多种方式录入新植物 |
| P-002 | 植物详情查看 | P0 | 低 | 展示植物完整档案信息 |
| P-003 | 编辑植物信息 | P0 | 低 | 修改植物的各项属性 |
| P-004 | 删除植物 | P0 | 低 | 移除植物档案（支持恢复） |
| P-005 | 生长阶段管理 | P0 | 中 | 记录和切换植物生长阶段 |
| P-006 | 健康状态标记 | P0 | 低 | 标记当前健康状态 |
| P-007 | 照片时间轴 | P1 | 中 | 记录植物生长过程影像 |
| P-008 | 养护记录 | P0 | 中 | 记录每次护理操作 |
| P-009 | 植物列表与筛选 | P0 | 中 | 多维度筛选和排序 |
| P-010 | 批量操作 | P2 | 高 | 批量编辑/提醒/状态更新 |
| P-011 | 数据导出 | P2 | 中 | 导出为PDF/Excel格式 |
| P-012 | 植物搜索 | P0 | 低 | 快速查找特定植物 |

#### 3.1.3 详细功能规格

##### **P-001 添加植物**

**功能描述**：用户可以通过多种方式将新种植的植物录入系统。

**输入条件**：
- 用户已登录
- 当前植物数量 < 上限（普通用户50株，VIP用户500株）

**业务规则**：

1. **添加方式**（支持以下一种或多种）：
   - **手动输入**：用户自行填写植物品种名称
   - **拍照识别**：调用AI图像识别接口，自动识别品种
   - **扫码添加**：扫描购买时的二维码（需商家合作）
   - **从推荐库选择**：浏览热门/推荐植物品种库

2. **必填字段**：
   - 植物名称/昵称（用户自定义，如"我家的小薄荷"）
   - 品种（从标准品种库选择或AI识别）
   - 种植日期（默认今天，可修改）

3. **选填字段**：
   - 来源（种子/扦插/购买/赠送）
   - 购买价格
   - 阳台朝向（首次使用必填，后续默认）
   - 摆放位置
   - 盆器类型和尺寸
   - 土壤类型
   - 备注

4. **自动填充逻辑**：
   - 选择品种后，自动填充该品种的基础养护模板
   - 根据种植日期和品种特性，自动计算当前生长阶段
   - 自动生成首条养护提醒（基于品种默认周期）

**前置条件**：无

**后置条件**：
- 植物档案创建成功，分配唯一ID
- 生成默认养护计划
- 如果开启了推送通知，发送"添加成功"确认消息

**异常处理**：
- 品种名称模糊时：展示候选列表供用户选择
- 网络异常时：本地缓存，网络恢复后同步
- 超出数量上限时：提示升级VIP或删除旧植物

**界面原型要点**：
- 顶部Tab切换：手动添加 / 拍照识别 / 品种库浏览
- 表单采用分步式（Step 1: 基本信息 → Step 2: 环境信息 → Step 3: 确认）
- 拍照识别页面：相机取景框 + 实时识别提示 + 结果确认
- 底部显示"保存"按钮，填写进度实时保存草稿

**验收标准**：
- [ ] 支持至少3种添加方式
- [ ] 必填字段校验完整，错误提示清晰
- [ ] 拍照识别响应时间<3秒
- [ ] 草稿自动保存，退出后可继续编辑
- [ ] 添加成功后跳转到植物详情页

##### **P-002 植物详情查看**

**功能描述**：展示单株植物的完整档案信息，包括基本信息、生长状态、养护记录、照片时间轴等。

**页面结构**（从上到下）：

1. **头部区域**：
   - 植物主图（支持点击查看大图）
   - 植物昵称 + 品种学名
   - 健康状态标签（颜色编码：绿色=健康，黄色=亚健康，红色=异常）
   - 当前生长阶段徽章
   - "编辑"快捷入口

2. **关键指标卡片**（横向滑动）：
   - 已养护天数：XX天
   - 下次浇水：X天后（倒计时）
   - 生长高度：XXcm（较上周+Xcm）
   - 护理次数：浇水X次 / 施肥X次

3. **Tab标签页**：
   - **概览Tab**：
     - 基本信息（品种、科属、来源、种植日期）
     - 环境信息（朝向、位置、盆器、土壤）
     - 最近一次养护记录摘要
   
   - **养护日历Tab**：
     - 日历视图：标记已完成任务（✓）和待办任务（⏰）
     - 列表视图：按时间倒序的养护记录列表
     - 快捷操作："记录浇水"、"记录施肥"按钮
   
   - **生长轨迹Tab**：
     - 时间轴照片墙（按月份分组）
     - 支持对比查看（选择两张照片左右对比）
     - 关键节点标注（发芽/开花/结果日期）
   
   - **AI诊断记录Tab**（如有）：
     - 历史识别记录列表
     - 每条记录包含：日期、图片缩略图、诊断结果、处理状态

**交互细节**：
- 下拉刷新：同步最新数据
- 长按主图：弹出菜单（更换封面/查看原图/删除）
- 点击养护记录：展开查看详情（时间、备注、图片）
- 左滑删除：养护记录支持左滑删除（二次确认）

**性能要求**：
- 页面加载时间<1.5秒（含图片懒加载）
- 照片时间轴采用虚拟滚动，避免内存溢出
- 离线时可查看最近缓存的数据

**验收标准**：
- [ ] 所有信息准确展示，无数据丢失
- [ ] Tab切换流畅，无明显卡顿
- [ ] 照片加载采用渐进式（先模糊后清晰）
- [ ] 支持离线查看基础信息（最近一次缓存）

##### **P-005 生长阶段管理**

**功能描述**：跟踪植物的生长周期，支持用户手动或系统自动切换生长阶段。

**预定义生长阶段**（根据植物类型不同有所差异）：

| 阶段代码 | 阶段名称 | 典型特征 | 默认持续时间 |
|---------|---------|---------|-------------|
| `germination` | 发芽期 | 种子破土，子叶展开 | 7-21天 |
| `growth` | 生长期 | 快速长高，叶片增多 | 30-90天 |
| `flowering` | 开花期 | 出现花蕾，花朵绽放 | 14-60天 |
| `fruiting` | 结果期 | 花谢后结果（如适用） | 30-90天 |
| `dormant` | 休眠期 | 生长缓慢或停止（冬季） | 60-120天 |

**业务规则**：

1. **阶段切换触发方式**：
   - **手动切换**：用户在植物详情页点击当前阶段，选择新阶段
   - **系统提示**：基于品种平均周期，到达预期时间后弹窗提示"是否进入下一阶段？"
   - **AI识别**：通过拍照识别植物形态特征，AI建议阶段（可选）

2. **阶段切换校验**：
   - 不允许跳过中间阶段（除非特殊情况备注原因）
   - 从休眠期切回生长期需要确认（可能是误操作）
   - 每次切换必须记录变更时间和备注

3. **阶段相关联动**：
   - 切换阶段后，自动调整养护方案参数
     - 例：从生长期→开花期，施肥频率增加，磷钾肥比例提高
   - 生成阶段里程碑事件，可在时间轴查看
   - 发送阶段变化通知（如果用户开启）

**数据模型**：
```json
{
  "plantId": "plant_001",
  "stageHistory": [
    {
      "stage": "germination",
      "startDate": "2024-03-15",
      "endDate": "2024-04-01",
      "durationDays": 17,
      "notes": "3月20日首次发芽",
      "changedBy": "user", // user | system_prompt | ai_suggestion
      "createdAt": "2024-03-15T10:00:00Z"
    },
    {
      "stage": "growth",
      "startDate": "2024-04-01",
      "endDate": null,
      "durationDays": null,
      "notes": "长出第6片真叶，进入快速生长期",
      "changedBy": "user",
      "createdAt": "2024-04-01T09:30:00Z"
    }
  ],
  "currentStage": {
    "stage": "growth",
    "sinceDate": "2024-04-01",
    "daysInCurrentStage": 44,
    "expectedNextStage": "flowering",
    "estimatedTransitionDate": "2024-06-15"
  }
}
```

**界面交互**：
- 详情页头部显示当前阶段图标+名称
- 点击阶段名称弹出"阶段选择器"（横向时间轴样式）
- 已完成的阶段灰色显示，当前阶段高亮，未来阶段虚线显示

**验收标准**：

**支持识别的病虫害类别**（MVP版本目标50类，后续扩展至200+）：

| 大类 | 包含的具体病害/虫害 | 典型特征 |
|------|-------------------|---------|
| **真菌病害** (15类) | 白粉病、霜霉病、锈病、灰霉病、叶斑病、炭疽病、根腐病等 | 白色粉末、黄色斑点、黑色霉层 |
| **细菌病害** (5类) | 软腐病、青枯病、溃疡病、叶枯病等 | 组织腐烂、维管束变色 |
| **病毒病害** (3类) | 花叶病毒、皱缩病毒、黄化病毒 | 花叶、皱缩、矮化 |
| **生理性病害** (7类) | 缺素症（N/P/K/Fe/Mg）、药害、肥害、日灼、冻害 | 黄叶、焦边、生长不良 |
| **刺吸式害虫** (8类) | 蚜虫、红蜘蛛、介壳虫、粉虱、蓟马等 | 叶片卷曲、蜜露、蜕皮 |
| **咀嚼式害虫** (7类) | 菜青虫、蜗牛、蛞蝓、跳甲、蛾类幼虫等 | 缺刻孔洞、粪便、虫体 |
| **地下害虫** (5类) | 蝼蛄、蛴螬、地老虎、线虫等 | 根系损伤、植株萎蔫 |

**识别流程**：

```
[用户触发拍照]
    ↓
[图像采集]
    ├── 相机取景（实时帧预览）
    ├── 网格辅助线（帮助对焦）
    └── 拍摄引导提示（"请靠近病灶部位"、"光线充足时效果更好"）
    ↓
[图像预处理]（服务端）
    ├── 质量检测（模糊/过暗/过曝检测，不合格则提示重拍）
    ├── 感兴趣区域裁剪（自动定位病灶区域）
    ├── 归一化（resize至224x224或448x448）
    └── 数据增强（亮度/对比度标准化）
    ↓
[模型推理]
    ├── 主模型：EfficientNet-B4 (预训练ImageNet + Fine-tune)
    ├── 输出：Top-3预测结果 + 置信度
    └── 推理时间：<500ms (GPU) / <2s (CPU优化)
    ↓
[后处理与整合]
    ├── 置信度阈值判断（>70%直接输出，40%-70%提示不确定，<40%建议人工）
    ├── 匹配知识库获取解决方案
    ├── 生成结构化报告
    └── 记录识别日志（用于模型迭代）
    ↓
[结果展示]
    ├── 诊断结论（病害名称 + 拉丁名 + 置信度）
    ├── 病害科普（简介、发病规律、传播途径）
    ├── 解决方案（分步骤、配图说明）
    ├── 预防措施（长期建议）

```

### 6.2 内部API规范

**RESTful API 设计原则**：

- **Base URL**: `https://api.plantbrain.com/v1`
- **认证方式**: `Authorization: Bearer <JWT_TOKEN>`
- **请求格式**: JSON (`Content-Type: application/json`)
- **响应格式**: 统一封装

**统一响应结构**：
```json
{
  "code": 0,           // 0=成功，非0=错误码
  "message": "success", // 错误信息（成功时可为空）
  "data": {},          // 业务数据
  "timestamp": 1715856000000,
  "requestId": "req_abc123" // 请求追踪ID
}
```

**分页规范**：
```json
{
  "code": 0,
  "data": {
    "list": [],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 100,
      "totalPages": 5
    }
  }
}
```

**错误码规范**：

| 错误码范围 | 含义 | 示例 |
|-----------|------|------|
| 0 | 成功 | - |
| 1000-1999 | 参数错误 | 1001=必填参数缺失 |
| 2000-2999 | 认证错误 | 2001=Token无效/过期 |
| 3000-3999 | 业务逻辑错误 | 3001=植物不存在 |
| 4000-4999 | 权限不足 | 4001=无权操作此资源 |
| 5000-5999 | 第三方服务错误 | 5001=AI服务超时 |
| 9000-9999 | 系统内部错误 | 9001=数据库连接失败 |

### 6.3 WebSocket协议（群聊）

**连接地址**: `wss://ws.plantbrain.com/chat`  
**认证**: 连接时传递Token参数  
**心跳**: 每30秒发送ping，服务端回复pong  
**消息格式**:

```json
{
  "type": "message",  // message/system/heartbeat/ack
  "groupId": "group_001",
  "senderId": "user_123",
  "contentType": "text", // text/image/audio/plant_card/location
  "content": "大家好",
  "clientId": "msg_abc456", // 客户端消息ID（去重用）
  "timestamp": 1715856000000,
  "metadata": {
    "mentionedUsers": ["user_789"], // @的用户列表
    "replyToMessageId": "msg_xyz" // 回复的消息ID（可选）
  }
}
```
